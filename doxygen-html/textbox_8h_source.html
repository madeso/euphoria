156  /* An utility function that can be used to create a tree graph rendering from a structure.
157  *
158  * Parameters:
159  * e: The element that will be rendered.
160  * Possibly some user-defined type that represents a node in a tree structure.
161  * maxwidth: The maximum width of the resulting box in characters.
162  * to_string: A functor of type std::string(const T&).
163  * It renders the given element into an 1D string.
164  * Note that the string must not contain multibyte characters,
165  * because size() will be used to determine its width in columns.
166  * count_children: A functor of type std::pair<ForwardIterator,ForwardIterator>(const T&).
167  * It returns a pair of iterators representing the range of children
168  * for the given element.
169  * create_tree_graph will call itself recursively for each element in this range.
170  * oneliner_test: A functor of type bool(const T&).
171  * If the result is true, enables simplified horizontal topology.
172  * simple_test: A functor of type bool(const T&).
173  * If the result is true, enables very simplified horizontal topology.
174  * separate1st_test: A functor of type bool(const T&).
175  * If the result is true, create_tree_graph() will always render
176  * the first child alone on a separate line, but the rest of them
177  * may get rendered horizontally.
178  *
179  * @param margin the spacing between children
180  * @param firstx the first child offset
181  *
182  * Topology types:
183  *
184  * Vertical:
185  *
186  * element
187  * ├─child1
188  * ├─child2
189  * └─child3
190  *
191  * Horizontal:
192  *
193  * element
194  * └─┬─────────┬─────────┐
195  * child1 child2 child3
196  *
197  * Simplified horizontal:
198  *
199  * element──┬───────┬───────┐
200  * child1 child2 child3
201  *
202  * Very simplified horizontal:
203  *
204  * element──child1
205  *
206  * The vertical and horizontal topologies are automatically chosen
207  * depending on the situation compared to the maxwidth parameter,
208  * and according to the constraint given by separate1st_test().
209  *
210  * Simplified topology will be used if oneliner_test() returns true,
211  * separate1st_test() returns false,
212  * all children fit on one line,
213  * and very simplified topology is not used.
214  *
215  * Very simplified topology will be used if oneliner_test() returns true,
216  * separate1st_test() returns false,
217  * simple_test() returns true,
218  * there is only 1 child,
219  * and it fits on one line.
220  */
